## 学习方法

这个仓库不是“让 AI 生成完整项目”的提示词集合。每个主题都使用同一个闭环：

1. **理解问题**：它解决什么问题，为什么需要它。
2. **先做设计**：写清输入、输出、流程、边界和验收标准。
3. **最小实现**：自己完成核心链路，AI 可以解释、评审和辅助编码。
4. **测试验证**：用正常、异常和边界用例证明它可用。
5. **定位问题**：先提出排查假设，再让 AI 帮助补充。
6. **总结表达**：记录决策、踩坑和改进，形成可用于面试的项目经验。

AI 可以负责资料检索、方案比较、代码样板、评审和调试辅助；你必须负责需求拆解、技术选择、核心代码理解、测试和最终判断。

建议每次学习都留下四类记录：

```text
我理解了什么：
我为什么这样设计：
我遇到了什么问题，如何验证：
如果重新做一次，我会改什么：
```

## 项目主线

### 项目一：AI Chat Playground

目的不是重复做聊天 UI，而是掌握所有 AI 应用共用的基础链路。

必须完成：

- FastAPI 路由、请求校验、统一异常响应
- 可替换的模型客户端与环境变量配置
- 非流式响应和 SSE 流式响应
- System / User / Assistant 消息与多轮上下文
- 超时、错误映射、基础日志和接口测试
- Vue 端流式消费、Markdown 渲染和中止生成

暂时不做用户系统、复杂数据库和精致 UI。

### 项目二：企业知识库 RAG

```text
文件上传 -> 解析 -> 清洗 -> 切片 -> Embedding -> 向量存储
                                                 |
用户问题 -> 查询改写 -> 检索 -> Rerank -> 上下文构造 -> LLM -> 答案与引用
```

核心训练点：可观察的数据处理链路、检索质量、引用溯源、权限过滤和评测，而不是只会调用 LangChain。

### 项目三：业务 Agent 自动化平台

选择一个具体业务场景，例如客户分析、订单处理或报表生成。实现工具白名单、结构化参数、执行轨迹、失败重试、人工确认与状态持久化。

## 当前阶段怎么做

Python 基础已经完成，现在进入阶段 1：

1. 先完成 FastAPI 的路由、Pydantic 校验、依赖注入、异常处理和异步基础。
2. 用独立脚本跑通一次 LLM 调用，观察原始请求、响应和 Token 用量。
3. 将模型调用封装到 service 层，完成 `GET /health` 与 `POST /api/chat`。
4. 实现 `POST /api/chat/stream`，理解模型流、SSE 和前端流是三层不同边界。
5. 补充超时、错误映射、日志和测试。
6. 接入 Vue 前端，只做最小聊天体验。
7. 完成阶段复盘，通过验收后再进入 RAG。

详细任务、代码骨架、练习和验收题见：[文档/03-FastAPI与LLM应用基础.md](./文档/03-FastAPI与LLM应用基础.md)。

## 每周投入建议

- 工作日：每天 60-90 分钟
- 周末：4-6 小时
- 每周总投入：8-12 小时

单次 90 分钟建议：

```text
15 分钟：明确今天的问题和验收条件
20 分钟：阅读官方资料并做最小实验
45 分钟：实现、测试、排查
10 分钟：记录结论和下一步
```

如果某个模块你已经熟悉，就用验收题快速跳过；不要为了填满一周而重复学习。

## 最终验收标准

完成路线后，你应该能够：

- 独立拆解 AI 应用需求并划分前端、业务服务、模型与数据边界
- 独立开发 FastAPI AI 服务和 Vue AI 产品交互
- 接入并替换 OpenAI、DeepSeek、Qwen 等模型服务
- 实现流式输出、多轮上下文、结构化输出与工具调用
- 实现文档解析、切片、Embedding、检索、引用和 RAG 评测
- 设计有权限、确认、重试和审计能力的 Agent 工作流
- 使用 Docker Compose 部署并通过日志、Tracing 和指标定位问题
- 不依赖背诵框架 API，能根据原理选择方案并验证结果

## 暂不作为主线

- 从零训练大模型
- 深挖 Transformer 数学推导
- PyTorch 训练与 CUDA 优化
- 论文复现和分布式训练
- 追逐大量 Agent 框架

这些内容不是没有价值，而是与当前“尽快具备企业 AI 应用交付能力”的目标优先级不匹配。
