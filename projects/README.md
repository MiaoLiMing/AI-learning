# 项目作品集规划

## 项目优先级

必须完成：

1. 企业知识库 RAG 系统
2. 业务 Agent 自动化平台

有余力再做：

3. AI 工作台 / Prompt 调试台
4. ChatBI 数据问答
5. 多模型网关

## 项目一：企业知识库 RAG 系统

### 业务背景

企业内部文档、制度、产品资料、售后手册分散，员工和客服查询效率低。系统通过上传文档、向量检索和大模型问答，帮助用户快速获得有引用来源的答案。

### 核心功能

- 知识库创建
- 文档上传
- 文档解析
- 文本切分
- Embedding 生成
- 向量检索
- RAG 问答
- 引用来源展示
- 对话历史
- 管理后台

### 技术栈

```text
前端：Vue 3 + TypeScript
后端：FastAPI
数据库：PostgreSQL + pgvector
缓存：Redis
模型：DeepSeek / Qwen
部署：Docker Compose
```

### 可加分功能

- 混合检索
- rerank
- 多知识库权限
- Token 成本统计
- 检索评测集
- 文档更新后重新向量化

### README 必须包含

- 项目介绍
- 架构图
- 功能截图
- 启动方式
- 环境变量说明
- API 示例
- RAG 流程说明
- 已知问题和优化方向

## 项目二：业务 Agent 自动化平台

### 业务背景

销售、客服、运营等岗位经常需要跨系统查询信息、整理内容、生成邮件、创建待办。Agent 平台通过工具调用和工作流编排，将多步骤任务自动化。

### 核心功能

- Agent 任务创建
- 工具调用
- 多步骤工作流
- 执行轨迹
- 工具调用日志
- 人工确认
- 失败重试
- 任务历史

### 示例业务场景

销售助理 Agent：

```text
用户输入：帮我给客户 A 生成一封报价跟进邮件。

Agent 步骤：
1. 查询客户信息
2. 查询历史订单
3. 查询最近沟通记录
4. 生成邮件草稿
5. 等待人工确认
6. 创建待办或发送邮件
```

### 技术栈

```text
前端：Vue 3 + TypeScript
后端：FastAPI
Agent：LangGraph
数据库：PostgreSQL
缓存：Redis
模型：DeepSeek / Qwen
部署：Docker Compose
```

### 可加分功能

- Agent 调试台
- 工具权限配置
- Prompt 版本管理
- 多 Agent 模板
- 人工确认节点
- 执行中断和恢复
