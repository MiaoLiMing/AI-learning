# 你真正需要掌握什么？

以 RAG 为例：

## 不需要：

背 LangChain API。

例如：

```python
RecursiveCharacterTextSplitter()
```

这种不用背。


---

## 必须理解：

### 1. 为什么切片？

因为：

LLM上下文有限。


### 2. 为什么Embedding？

因为：

文本无法直接计算相似度。

需要：

```
文字

↓

向量

↓

距离计算
```


### 3. 为什么Vector DB？

因为：

普通数据库：

```
where content like '%退款%'
```

无法理解语义。


向量：

```
退款
退钱
费用返还

距离接近
```


### 4. 为什么会幻觉？

因为：

LLM不是数据库。

它是在预测文本。
