# 面试与简历准备

## 1. 简历定位

不要写“正在学习 AI”。

建议标题：

```text
AI 应用全栈开发工程师
```

或：

```text
大模型应用开发工程师 / RAG & Agent 工程师
```

## 2. 个人简介模板

```text
具备 Vue、TypeScript、Node.js 前端与全栈开发经验，正在转向 AI 应用开发方向。
熟悉 FastAPI、LLM API、RAG、Agent、向量数据库和 Docker 部署。
能够独立完成 AI 聊天系统、企业知识库问答、业务 Agent 自动化平台等应用的前后端开发与部署。
```

## 3. 项目一：企业知识库 RAG 系统

简历写法：

```text
企业知识库 RAG 系统
- 实现文档上传、解析、切分、Embedding、向量检索和引用溯源
- 基于 PostgreSQL + pgvector 构建知识库检索服务
- 接入 DeepSeek / Qwen，支持流式问答和多轮对话
- 前端实现知识库管理、对话历史、引用片段展示和 Markdown 渲染
- 增加日志记录、Token 成本统计、错误重试和 Docker Compose 部署
```

面试讲法：

- 先讲业务场景：解决企业内部文档难查、客服重复回答的问题。
- 再讲架构：前端、后端、向量库、模型服务、任务队列。
- 再讲难点：文档切分、召回质量、幻觉控制、引用溯源。
- 最后讲优化：混合检索、rerank、成本统计、评测集。

## 4. 项目二：业务 Agent 自动化平台

简历写法：

```text
业务 Agent 自动化平台
- 基于 LangGraph 实现多步骤 Agent 工作流
- 封装客户查询、订单查询、邮件生成、待办创建等业务工具
- 支持工具调用日志、执行轨迹、任务状态持久化和失败重试
- 对高风险操作加入人工确认机制
- 前端实现 Agent 调试台、执行步骤展示和工具调用结果展示
```

面试讲法：

- 先讲 Agent 与普通聊天机器人的区别。
- 讲清楚工具调用链路：模型决定调用工具，后端执行工具，结果返回模型，模型继续推理。
- 讲权限控制和人工确认。
- 讲失败重试和执行日志。

## 5. 高频面试题

### RAG 类

1. RAG 是什么？为什么不直接把文档塞进 Prompt？
2. Chunk 怎么切？切大了和切小了分别有什么问题？
3. Embedding 是什么？向量检索为什么能找相似内容？
4. 如何提升召回质量？
5. 什么是 rerank？
6. 如何做引用溯源？
7. RAG 为什么还会幻觉？怎么降低？
8. 怎么评估一个 RAG 系统效果？

### Agent 类

1. Agent 和 Chatbot 有什么区别？
2. Function Calling 的完整流程是什么？
3. 工具调用怎么做权限控制？
4. 如何避免 Agent 误操作？
5. 人工确认怎么设计？
6. Agent 执行失败怎么办？
7. 为什么需要工作流框架？
8. LangChain 和 LangGraph 的区别是什么？

### 后端工程类

1. SSE 和 WebSocket 的区别是什么？
2. 流式输出怎么实现？
3. 如何统计 Token 成本？
4. 如何做模型请求失败重试？
5. 如何避免接口被刷爆？
6. 文件上传后如何异步解析？
7. Docker Compose 怎么组织服务？
8. 如何管理模型 API Key？

### 安全类

1. 什么是 Prompt Injection？
2. 如何防止用户让模型泄露系统提示词？
3. RAG 中恶意文档可能造成什么问题？
4. 工具调用为什么要做白名单？
5. 如何处理敏感信息？

## 6. 投递策略

优先投：

- 20-200 人的 SaaS 公司
- 做企业服务的中小公司
- 有内部知识库、客服、CRM、ERP、OA 场景的公司
- 招“AI 应用开发”而不是“算法研究”的公司
